Задача 2
Делаем имитацию системы управления пользователями в очень сокращенном виде.

1. у нас должна быть одна страница
2. выводим таблицу с пользователями (детали ниже)
4. выводим над и под таблицей блок со следущими моментами,
	a. кнопка add
	b. селектбокс котрый имеет следующие элементы (1. Please select, 2. Set active, 3. Set not active, 4. Delete)
	c. кнопку OK

*Таблица

1. поле чекбокс (логика работы стандартная, нажал и все выбрались, отжал какой-то то и отжался верхний так как не все выбраны), это чек бокс для групповх действий
2. Name (First Name Last Name)
3. Status (выводим зеленый кружок если актив и серый если нет)
4. Role (admin/user)
5. Options состоит с двух кнопок
	а. Edit (иконки)
	b. Delete (иконки)

Таблицу берем готовую бутстрап, все иконки используем по умолчанию FontAwesone
Примеры взять тут https://www.bootdey.com/ или https://bootsnipp.com/

*форма добавления/правок (модальное окно бутстрап)
должно быть ОДНО окно на добавление и правку
состоит со следующих полей
1. First Name (текст)
2. Last Name (текст)
2. Status (чекер как у гугла, полоска с кружечком, https://www.cubum.ru/wp-content/uploads/2018/05/checkbox-html-primery-krasivyh-knopok-na-chistom-css-sozdanie-akkordeona-poluchenie-znacheniy-iz-form-s-pomosch-yu-php-i-jquery-3.jpg)
3. Role (селект бокс)

*конфирм окна
они должны быть где логически они нужны, например при удалении, или групповых операциях (когда не выбраны все и т.д.)
конфирм окна должны выглядеть также используя модальные формы, не используем стантардные конфирм браузера.
